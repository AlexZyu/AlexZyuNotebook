# 一、简历

# 1. 在项目《安全应急产业专利被引与专利权放弃之间的相关性分析》中，你做了什么？你觉得最难的地方是什么？能不能给我描述下你们的项目？（选题到结论）

  1. 老师好，我先给您介绍一下这个项目的核心结论：发生了被引的专利更倾向于放弃专利权。其研究意义在于，为“专利被引”现象的情报学意义增添了新内容，也给专利评价提供了新视角（法学、创新经济学）。
  2. 该项目是《信息分析》课程的延续，选题来自老师李睿教授，我主要是参与了样本集和验证集数据的收集，以及撰写log-linear对数线性模型和交叉列联表相关性分析的初稿。当然里面还有一些零碎的部分，比如第一部分的文献综述
  3. 该项目中最难的部分在于选题，当然这一点由老师把关，也算进行得顺利。其次，对我个人而言是第一次接触专利文献计量，由于“专利被引频次”是高度依附于时间的非随机线性序列，在要避免时间这一潜变量可能造成的虚假相关，选择合适的检验方法花了我很大的精力。

提示🎗️

对数线性模型

（1）作用：综合运用方差分析和逻辑回归中的建模方法，应用于纯粹定类变量之间，系统评价各变量间关系和交互作用大小的多元统计方法；

（2）优缺点：可以直接分析各种类型的分类变量，对于名义变量，也不需要事先建立哑变量，可以直接分析变量的主效应和交互效应。对数线性模型不仅可以解决卡方分析中常遇到的高维列联表的“压缩”问题，又可以解决logistic回归分析中多个自变量的交互效应问题。

提示🎗️

在创新经济学鼻祖 J. A. Schumpeter 看来，创新的手段是“毁损”旧技术，即“创新”的新组合将会打破过去旧的、低效的工艺与产品，创新的过程是“旧组合-分拆-新组合”的过程;

创造性毁损理论（creative destruction） 认为，社会经济发展的过程是新技术不断破坏旧技术的过程，即企业不断研发效用更好的新产品以淘汰旧产品的过程

精髓是“强调核心竞争优势的获取与保持”

心理学认为，逃是本能，战是选择。

# 2. 为什么要做这个索引？

  1. 老师好，当初选择做这个索引是从自身兴趣+现实条件出发的。因为在去年7月份做了科学计量相关的研究——《安全应急产业专利被引与专利权放弃之间的相关性分析》有一定知识储备的同时，确实对这个文献计量、科学计量方向感兴趣。
  2. 另一个原因是现在大多院校的博士学位论文不会收录进网络数据库了，当时选论文的时候正好在知网上检索到您的论文，第一感觉是做得很厉害、符合我的选择预期，所以就选了。

# 3. 你为什么会选我？

  1. 对老师您在科学计量方面的研究既是有兴趣、又是敬佩。我想，要在一个感兴趣的学术领域深入探索有一个好老师是必要的。
  2. 其次是，我也了解到老师您刚加入武大不久，在很短的时间内已经有了丰硕地成果，将来也需要做更密集更深入的研究，这与我自己想多在学术上做实事的预期相符。

# 2. 图书馆学和信息资源管理的区别和联系

提示🎗️

英国图书馆战略规划中说到：“我们所处的环境在过去的二十年里发生的变化超过了过去两百年的变化，特别是技术发展的驱动。”初景例教授后来又加了一句，未来五年的变化可能会让过去二十年的变化几乎可以忽略不计。

  1. 信息资源管理的外延大于图书馆学，如果对这两个学科进行概念的划分与概括，信息资源管理是图书馆学的上位概念，而图书馆学继承了其全部内涵。
  2. 信息资源管理以“数据思维、系统思维、用户思维、技术思维、管理思维”五维一体为指导，强化信息资源认知观，打造数据技能链，结合公共信息资源开发与利用场景，培养“宽口径、厚基础、多层次”的复合型信息管理专门人才。
  3. 图书馆学运用现代化技术手段组织、检索、分析、评价和开发利用信息的能力，能在图书馆、信息服务机构和各类企事业单位的信息部门从事信息服务及管理工作，适应经济和社会发展需要的厚基础、宽口径、高素质、强能力的创造、创新、创业型复合人才。
  4. 进入到数智时代，我们的图书馆学是信息资源管理框架下的图书馆学，将更广阔的研究外延纳入学科范围，引入了更多的计算、研究方法，让图书馆学的研究不再局限于“图书馆”这一实体。
  5. 从今年我们一级学科的改名的角度来看，可能做传统图书馆学采访、编目、组织的学者会觉得不舒服，但对我们学科探索新边界、进入学科范式的转换是一次新的机会。

# 3. 本科是信息资源管理

  1. 我本科的信息资源管理是由信息管理与信息系统和信息资源两个专业合并而来，因此也继承了这两个专业的培养方案，既有信息系统的知识也有传统图书馆信息资源建设、信息服务的内容。
  2. 以较为宽泛的通识教育为主。因为川大特别强调培养跨学科、多层次的人才，自己也修读了除本专业外包括法学、传播学、历史学在内的一些人文通识课。
  3. 师资队伍的配置也是兼顾了信管和信资两方向，学生可以根据自己的兴趣来深入学习。
