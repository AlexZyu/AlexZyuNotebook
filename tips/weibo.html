# 数据获取-微博weibo爬虫

# weibo-search包

获取脚本：

$ git clone https://github.com/dataabc/weibo-search.git

# 配置

本程序的所有配置都在setting.py文件中完成，该文件位于weibo-search\weibo\settings.py。

提示🎗️

所有配置方式参考之前的代码即可。

注意❗

很重要的一步是要记得更改cookie。那么如何更改cookie呢？

  1. 用Chrome打开 微博open in new window
  2. 点击"立即登录"，完成私信验证或手机验证码验证，进入新版微博
  3. 按F12打开开发者工具，在开发者工具的Network->Name->weibo.cn->Headers->Request Headers，找到"Cookie:"后的值，这就是我们要找的cookie值，复制即可。

# 运行

  1. 首先打开Anaconda终端。

注意❗

使用windows自带的终端是不行的。

  1. 进入代码根目录。
  2. 运行以下代码：
$ scrapy crawl search -s JOBDIR=crawls/search

其实只运行scrapy crawl search也可以，只是上述方式在结束时可以保存进度，下次运行时会在程序上次的地方继续获取。注意，如果想要保存进度，请使用Ctrl + C一次，注意是一次。按下Ctrl + C一次后，程序会继续运行一会，主要用来保存获取的数据、保存进度等操作，请耐心等待。下次再运行时，只要再运行上面的指令就可以恢复上次的进度。

# 用户词典使用方式

根据具体文本调整词典

  • add_word_list.txt作用是告诉jieba “自定义词汇” 是一整个词语，不要分成两个词语，并且告诉jieba该词的词性。

  • 词典以如下形式书写：

自定义词汇 词性
如：
成都疫情 n
音乐节 n
马思唯 n
  • 词汇可以是中文，也可以是英文；词汇顶格，按space空格后写词汇的词性。
  • 词性名称对应表查询：词性名称对应表-CSDNopen in new window

# 不区分同义词

  • 直接运行word.ipynb代码文件的wordfreq-没有同义词部分即可。
  • 输入“当前文件夹路径”时，采用绝对路径的形式。
  • 输入“要分析的文本”时，一般直接使用test.txt即可。把所有文本直接粘贴进去即可。
  • 最后会生成word_freq.xlxs文件，就可以了。

# 需合并同义词

  • 什么情况需要合并同义词？
    • “教师” vs “老师”
    • “核酸检测” vs “核酸”
    • ...
  • 如何合并？
    • 创建同义词表synonym_list.xlsx文件
    • 创建两列的内容，表头为origin和new
    • 下面的内容，要替换的词为origin，替换后的词为new
  • 使用wordfreq-合并同义词部分即可。

# 需要把多个下位词合并为一个上位词

  • 创建一个synonym.txt文件
    • 该文件中，一行为一个项目。每一行第一个词为要替换成的上位词，后面用space空格隔开无数个下位词。
    • 一个项目结束后敲enter换行。
  • 使用wordfreq-合并同义词2部分。

n_features = 1000 #提取1000个特征词语
tf_vectorizer = CountVectorizer(strip_accents = 'unicode',
                                max_features=n_features,
                                stop_words='english',
                                max_df = 0.5,
                                min_df = 10)
tf = tf_vectorizer.fit_transform(data.content_cutted)

这一串代码中：

  • n_features的数值可以根据需要自行修改。
n_topics = 4 #主题数
lda = LatentDirichletAllocation(n_components=n_topics, max_iter=50,
                                learning_method='batch',
                                learning_offset=50,
                                 doc_topic_prior=0.1,
                                 topic_word_prior=0.01,
                               random_state=0)
lda.fit(tf)

这一串代码中：

  • n_topics的数值可以根据需要自行修改。

提示🎗️

这里的 n_topic 的确定是一个关键问题，直接关乎到LDA模型聚类的可靠性，需要多次找寻，以求得最佳结果。

# 2.4 困惑度

使用困惑度（Perplexity）指标来评价该LDA模型。一般认为，困惑度越低，说明聚类的效果越好。

参考：折肘法+困惑度确定LDA主题模型的主题数_CSDNopen in new window

参考描述语言：

  • 在上图中，随着K（Number of Topic）值的增大，困惑度（Perplexity）逐渐减小。根据手肘法，并且当K约为 ? 的时候，存在一个显著的拐点：
  • 当K属于（1, K=?）时，曲线急剧下降；
  • 当K属于（K=?, 9）时，曲线再次上升。
  • 故拐点 ? 即为K的最佳值，因此在本文所选定的语料集中，LDA主题的生成数量选定为 ? 是合理的🎉
  • 值得注意的是，当随着 K> ? ，训练困惑度出现上升，这与样本空间和算法本身都有关联，属正常现象。
